🔌 API Documentation

REST API endpoints for PayVerify Nepal

📋 Base URL

http://localhost:5000/api/

🔐 Authentication

POST /login
Login to your account
POST /signup
Create a new account (age 25-55)

📤 Verification Endpoints

POST /verify
Upload a payment screenshot for verification
Request: multipart/form-data with 'screenshot' file
Response: JSON with verdict, confidence, flags, extracted fields
GET /health
Check if the API is running
Response: {"status": "ok", "model_loaded": true/false}

📊 Data Endpoints

GET /history
Get verification history (requires login)
GET /dashboard
Get dashboard statistics (requires login)
GET /export-history
Export history as CSV (requires login)
GET /download-pdf/{verification_id}
Download verification report as PDF

💻 Example Usage

# Upload a screenshot for verification curl -X POST -F "screenshot=@payment.png" http://localhost:5000/verify # Check API health curl http://localhost:5000/health # Export history (requires login) curl -X GET http://localhost:5000/export-history

🔒 Privacy & Ethics

  • 🔒 All processing is done locally — No data leaves your device
  • 👤 Age validation (25-55) for users
  • 🔑 Strong password requirements
  • 📊 False positive rate maintained below 5%
  • 📁 No real banking credentials stored
